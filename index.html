Definición y conceptos clave

¿Qué es?

Un servicio web es una funcionalidad accesible a través de una red (generalmente Internet) que permite la interoperabilidad entre aplicaciones. Expone operaciones (métodos) que otros sistemas consumen usando protocolos abiertos.

  • Proveedor: publica y ofrece el servicio.
  • Consumidor: cliente que envía solicitudes.
  • Contrato: reglas/formato de intercambio (p. ej., WSDL/JSON).
  • Mensajería: mensajes HTTP con cuerpo JSON o XML.

Principios de la arquitectura orientada a servicios (SOA)

      

        

          
    
            
  • Acoplamiento débil: cambios internos no rompen a los consumidores.
    • 
            
  • Reutilización: servicios compartidos por múltiples aplicaciones.
    • 
            
  • Contratos explícitos: interfaz estable y documentada.
    • 
            
  • Descubrimiento: posibilidad de encontrar servicios en un registro.
    • 
            
  • Composición: servicios se orquestan para procesos más grandes.
    • 
            
  • Autonomía/Estado mínimo: preferir operaciones idempotentes.

Tipos de servicios web: SOAP y REST

      

        

          
SOAP (Simple Object Access Protocol)

          
Protocolo basado en XML con mensajes estándar (sobre HTTP, SMTP, etc.). Utiliza contratos formales WSDL y define envelopes, cabeceras y cuerpo.

          
POST /ws/Estudiantes HTTP/1.1
Host: ejemplo.edu
Content-Type: text/xml; charset=utf-8

<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
<soap:Body>
<GetAlumno xmlns="http://ejemplo.edu/alumnos">
  <id>1023</id>
</GetAlumno>
</soap:Body>
</soap:Envelope>

          

            
            

          

        

        

          
REST (Representational State Transfer)

          
Estilo arquitectónico que expone recursos vía URLs y opera con HTTP y formatos ligeros como JSON. Métodos típicos: GET, POST, PUT, DELETE.

          
GET /api/estudiantes/1023 HTTP/1.1
Host: ejemplo.edu
Accept: application/json

          

            
            

              
              
              
            

            
            

          

        

      

      
Protocolos y estándares relacionados

      

        

          
HTTP

              
Protocolo de aplicación para la transferencia de documentos. Define métodos (GET/POST/PUT/DELETE), códigos de estado (200, 404, 500) y cabeceras.

              

                
                
                
                

              

        

        

          
XML & JSON

          
Formatos de intercambio de datos. XML es etiquetado y extensible; JSON es liviano y fácil de leer en JavaScript.

          
<alumno>\n  <id>1023</id>\n  <nombre>Ana</nombre>\n</alumno>

          
{
"id": 1023,
"nombre": "Ana"
}

        

        

          
WSDL & UDDI

          
WSDL describe operaciones y tipos de un servicio SOAP. UDDI es (o fue) un registro donde se publicaban/descubrían servicios.

          
<definitions name="AlumnoWS" targetNamespace="http://ejemplo.edu">\n  <message name="GetAlumnoRequest"/>\n  <portType name="AlumnoPort">\n    <operation name="GetAlumno"/>\n  </portType>\n</definitions>

          

            
            
            
            

          

        

      

    

  


  

    

      

        

          
Recuerda

          
SOAP = contratos formales y XML; REST = recursos sobre HTTP y JSON (usualmente). Ambos pueden coexistir según necesidades.
